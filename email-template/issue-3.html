Load, build and respond faster

What'd I say about JS chatter last week? :)

I got a bit style guide happy this week, but scaffolding techniques and libraries are required to building faster and more efficiently. This week is a bit of everything - load, build and respond faster.

Performance

Leverage browser caching - it's easier than you think

Run your website through a page speed tester like Google's and you'll likely get a crappy score on "Leveraging browser caching". If you don't know exactly what this means, don't worry, here's your how-to guide on getting closer to a page speed test perfect score.

http://fortheloveofseo.com/blog/performance/leverage-browser-caching-how-to-add-expires-headers/

The standard performance practice of compressing your website

A browser can receive and decompress a compressed package of your site faster than it can just receive the uncompressed version. This quick how-to has instructions for quickly adding a snippet to your .htaccess file. Need a deeper dive on gzipping vs. uncompressed?

http://www.feedthebot.com/pagespeed/enable-compression.html

SEO

Schema actions added - adding verbs to your SEO-friendly content

I mentioned Schema.org last week - and it was oh-so-timely. They've added "action" vocabulary which adds a whole new dimension: not only can you define nouns, but now you can define verbs.

http://terraces.wordpress.com/2014/04/16/the-new-schema-org-actions-what-they-mean-for-personalisation-on-the-web/

Learning

Why would you build a wiki and not a style guide for projects? Do so easily with Sass 3.3 maps.

Sass 3.3 has maps (aka. associative arrays) and a map-get() function for retrieving items in the map. The valuable use case: building a style guide for every project automatically can be as important as having a wiki for users. Need more reasons to do a style guide?

http://www.phase2technology.com/blog/exploring-maps-in-sass-3-3part-3-calling-variables-with-variables/

Libraries

Markdown in your stylesheet creates beautiful style guides

Create a full style guide inline in your .css, .sass, or .less files. Best part - code your HTML in markdown inline as well and let the Hologram listener/guard gem compile your stylesheets.

http://trulia.github.io/hologram/

Dev Tools

Opening doors in debugging with Chrome's Async Mode

By enabling Async mode in Chrome Dev Tools you unlock the whole call stack to be at your finger tips. Also, the last tip in the article should be the first on the topic of debugging in the console for debugging in the console: name functions, don't leave them as anonymous!

http://www.html5rocks.com/en/tutorials/developertools/async-call-stack/

Responsive

Go-to responsive image library revs to v2-alpha

I usually try to find articles that aren't necessarily mainstream, but this is a must-know. Picturefill v2, now in alpha, is the go-to library for responsive image handling.

http://scottjehl.github.io/picturefill/

UI/UX

You probably shouldn't be using infinite scroll

Sure, Pinterest, Twitter, and Facebook are good use cases for infinite scroll - the user actions are minor and don't take you away from the page. But most sites should absolutely not be using it.

http://geeks.bizzabo.com/10-reasons-not-to-use-infinite-scroll-on-your-website
