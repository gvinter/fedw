Saving time on all fronts... front-end dev pun intended

A few ways to save time:

  • Write code more efficiently and use modularity.
  • Use libraries more suited for the exact task at hand.
  • Avoid running redundant processes.
  • Use native/pre-existing libraries instead of building custom ones.
  • Don't let assets loading hinder loading other assets.
  • Use task runners to combine a full workflow into a single command.

These are some of the topics included this week. Which ones do you do?
Tell me about it.

Zurb has announced that later this year they will releases a "companion" framework to Foundation 5 called "Foundation for Apps", just like their Foundation for Email, aka. Ink. It's built initially with AngularJS but meant to be able to port over to other JS frameworks as well.
