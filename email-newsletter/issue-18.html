Libraries can broaden horizons to take on whole new project categories


Please excuse my tardiness this week. A bit of travel threw me off. We'll be back on schedule next Tuesday!

Now, let's get to it... the soul purpose of libraries is to give you a starting point on code, so you don't have to code certain features and functions from scratch. However, library-specific goals can include being faster, lighter, more usable/readable, or library add-ons (jQuery extensions, for example). Then there's a category of library that are meant to branch into whole new verticals - like the voice elements library I shared in June, Tracking.js and Heatmap.js do just that. They give us front-end developers the ability to take on whole new types of projects that we never would have touched before.

Libraries

A JS library for advanced face and eye tracking

I often avoid certain projects based on initial feature requests such as "I want advanced facial recognition". However, with libraries like Tracking.js, the amount of power put in the hands of a front-end developer is mounting!

trackingjs.com

This Heatmap library gives you access to incredibly valuable data

I think a lot about client projects and PROVING that a UI works. It's all about data, right? Well if bucking up $50/month for VWO and all the other features isn't what you want, and you need a custom solution for getting heatmap data from visitors, you're in luck here.

patrick-wied.at

Make a page load feel like a native experience

A page loading shouldn't feel like 100 parts eventually coming to your browser. smoothState allows you to control your page load experience and page transitions, and even has a cool built-in prefetch option for hover state of links, which is a lot like last week's prefetch article.

weblinc.github.io

Learning

A clever way of getting rid of a common pain in the butt

You built a beauitful responsive site. Your QA person opens it on their iPhone and there's horizonal overflow. WHAT?! 'No way', you think to yourself. But there it is on your iPhone too. Chris Coyier gets clever detecting and fixing this issue.

css-tricks.com

Z-Index organization for complicated UIs

I'll admit I often get further into a project than I should before I circle back to decide on a z-index ordering scheme. I'm going in this direction for now on from the start.

smashingmagazine.com

A system I love: Style Guide Driven Development

I'm not a web designer. I do UX designs/wireframes but, visual designs in PSD isn't my thing. For projects without designers I pick my fonts and color scheme, build a style guide, and then apply them to UX designs. Well, that's SDD, and you can learn more here.

webuild.envato.com

Resources

Sass/CSS code style is important, plan accordingly

Last week week we had Airbnb's JS style guide. Well, guideslines for CSS should be taken just as seriously. Here's a resource to improve code quality in your projects for CSS and CSS preprocessors (Sass, Less, etc.)

cssguidelin.es/

Frameworks

Famo.us gives devs the power of advanced visual UIs and interactions

I've been keeping my eye on Famo.us for a while. Although they're not v1 yet, I want this to now be on your radar as well, because their advanced rendering and 3D engines are amazing, and will certainly be something you should consider for upcoming advanced UI mobile and desktop projects.

famo.us
